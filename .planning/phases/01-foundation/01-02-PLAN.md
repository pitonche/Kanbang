**2. `src/components/Board.tsx`** -- Main board layout:

```tsx
import { Column } from "./Column";

export const COLUMNS = [
  { id: "inbox", label: "Inbox" },
  { id: "backlog", label: "Backlog" },
  { id: "in_progress", label: "In Progress" },
  { id: "needs_info", label: "Needs Info" },
  { id: "blocked", label: "Blocked" },
  { id: "done", label: "Done" },
] as const;

export type ColumnId = (typeof COLUMNS)[number]["id"];

export function Board() {
  return (
    <div className="flex gap-4 p-6 overflow-x-auto min-h-screen bg-board-bg">
      {COLUMNS.map((col) => (
        <Column key={col.id} id={col.id} label={col.label} />
      ))}
    </div>
  );
}
```

Key details:
- `COLUMNS` exported as `const` for type safety and reuse in later phases
- `ColumnId` type exported for schema/component alignment in later phases
- `overflow-x-auto` enables horizontal scrolling on narrow viewports
- `min-h-screen` makes the board fill the viewport height
- `gap-4` and `p-6` provide spacing between columns and board edges

**3. `src/components/Column.tsx`** -- Individual column:

```tsx
interface ColumnProps {
  id: string;
  label: string;
}

export function Column({ id, label }: ColumnProps) {
  return (
    <div className="flex flex-col w-72 shrink-0 rounded-lg bg-column-bg">
      <h2 className="px-3 py-2 text-sm font-semibold text-column-header uppercase tracking-wide">
        {label}
      </h2>
      <div className="flex-1 p-2 min-h-[200px]">
        <p className="text-sm text-empty-state text-center mt-8">
          No tasks yet
        </p>
      </div>
    </div>
  );
}
```

Key details:
- `w-72` fixed width (288px) per column -- standard Kanban column width
- `shrink-0` prevents columns from shrinking when viewport is narrow
- `min-h-[200px]` gives columns a visible drop zone height for future drag-and-drop
- `uppercase tracking-wide` on headers for visual distinction
- Empty state text centered for clean appearance
- `id` prop accepted but not used in Phase 1 -- needed in Phase 2 for data queries

All components in `src/components/` per user decision (flat folder structure).
